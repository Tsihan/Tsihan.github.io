修改ID放置策略

对于PropertyPlacementStrategy.java文件，修改内容如下：

    /**
     * 要修改
     *
     * @param vertex
     * @return
     */
    private int getPartitionIDbyKey(JanusGraphVertex vertex) {
        Preconditions.checkState(idManager != null && key != null,
            "PropertyPlacementStrategy has not been initialized correctly");
        assert idManager.getPartitionBound() <= Integer.MAX_VALUE;
        int partitionBound = (int) idManager.getPartitionBound();
        Iterator<JanusGraphVertexProperty> it = vertex.query().keys(key).properties().iterator();
        final JanusGraphVertexProperty p = it.hasNext() ? it.next() : null;
        if (p == null) {
            return -1;
        }
        //这块可以修改 hashPid通过DGSS的方式获得
        // int hashPid = Math.abs(p.value().hashCode())%partitionBound;
        int hashPid;
        int tempResult = (testGSS.nodeDegreeQuery((String) p.value(), 0) + testGSS.nodeDegreeQuery((String) p.value(), 1));
        if (tempResult >= 10000) {
            hashPid = random.nextInt(4);
        } else {
            if (tempResult % partitionBound > 4) {
                hashPid = tempResult % partitionBound;
            } else {
                hashPid = tempResult % partitionBound + 4;
            }

        }

        assert hashPid >= 0 && hashPid < partitionBound;
        if (isExhaustedPartition(hashPid)) {
            //We keep trying consecutive partition ids until we find a non-exhausted one
            int newPid = hashPid;
            do {
                newPid = (newPid + 1) % partitionBound;
                if (newPid == hashPid) //We have gone full circle - no more ids to try
                    throw new IDPoolExhaustedException("Could not find non-exhausted partition");
            } while (isExhaustedPartition(newPid));
            return newPid;
        } else return hashPid;
    }

由于java打包成jar包时外部的文本文件自动不被包含进入，当重新编译后需要手动将外部文件放到未来软件使用指定的位置。

生成可发布版本

提前需要安装好Maven（版本3）以及Java 8。建议将Maven配置文件添加国内镜像，例如：

    <mirror>
      <id>mirrorId</id>
      <mirrorOf>repositoryId</mirrorOf>
      <name>Human Readable Name for this Mirror.</name>
      <url>http://my.repository.com/repo/path</url>
    </mirror>

具体配置可点击此处。Ubuntu系统下该配置文件位于：/etc/maven/settings.xml或/usr/share/maven/conf/settings.xml。

解压源代码到指定位置，进入该目录下并输入以下终端指令：

 mvn clean install -Pjanusgraph-release -Dgpg.skip=true -DskipTests=true

如果对于源代码进行修改，对于新增加的文件，需要在头部增加以下内容：

否则无法通过编译，当新增加的代码块不满足编码规范但确保逻辑正确性时，终端命令修改如下，可以跳过代码规范检查：

#新增的末尾内容为跳过代码规范检查
mvn clean install -Pjanusgraph-release -Dgpg.skip=true -DskipTests=true -Dcheckstyle.skip
